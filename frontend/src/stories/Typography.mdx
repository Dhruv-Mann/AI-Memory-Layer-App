# Typography System

Our neo-brutalist typography system relies on stark contrasts, distinct hierarchies, and tight tracking to reinforce the data-dense, developer-tool aesthetic.

## Font Stacks
**Sans-serif (UI default):** Clean, system-native sans-serif fonts for optimal readability and zero-layout shift.
**Monospace (Code & Data):** Fixed-width fonts used for JSON chunks, memory IDs, and strict data displays.

## Guidelines & Pairings
- **Headings** should always use `font-bold` and `tracking-tight` with a `leading-compact` line height to feel dense and blocky.
- **Body text** should be `font-regular` and `leading-normal` to ensure readability over long passages.
- **Data labels** should be `text-xs`, `font-bold`, uppercase, and `tracking-wide`.
